What does "Microservices Architecture" mean?

A Microservice Architecture is an architecture that
allows to create a single page application by dividing
the overall application into smaller services. This means
each service is responsible of one thing. It is very
versatile but the problem comes when it is time to define
how big or small is the thing that it is responsible for.
Microservices is helpful and convenient because every service
is independently deployable and scalable and it is not needed
to do it for all the application but only that service.
On the contrary, a monolith architecture needs has all of its
functionality in a single process which means it needs to be
replicated completely. That doesn't mean a monolith is not
useful since it is great and easier for small applications.

not completely reloaded but just the parts that need to be reloaded.
In other words it "divides" the webpage into smaller parts named "views"

Explain how you can have more than one access key in your computer and how to use each one. Provi code or confguration examples.

To have more than one access key it is necessary to add to
the file that contains the keys the other keys that want to
be used. They must have a different label than "[default]"
and to use them it that label must be specified.

Example:

[default]
aws_access_key=*insert key here*
aws_secret_access_key=*insert key here*

[key2]
aws_access_key=*insert different key here*
aws_secret_access_key=*insert different key here*
